9. КЛАСС PLAYER (игрок)
python
class Player:
    def __init__(self, start_platform_y):
        self.width = 40          # Ширина персонажа
        self.height = 60         # Высота
        self.x = WIDTH//2 - 20   # Позиция по X (центр экрана)
        self.y = start_platform_y - self.height - 1  # Позиция по Y (над платформой)
        self.vel_y = 0           # Скорость по Y (падение/прыжок)
        self.vel_x = 0           # Скорость по X (движение влево/вправо)
        self.on_ground = True    # На земле ли?
        self.can_jump = True     # Может ли прыгать?
        self.invincible = True   # Неуязвим ли (после старта)
        self.invincible_timer = 100  # Таймер неуязвимости
Методы:

update(platforms)
Обновляет состояние игрока каждый кадр:
Уменьшает таймер неуязвимости
Применяет гравитацию к скорости (vel_y += GRAVITY)
Обновляет позицию (y += vel_y, x += vel_x)
Проверяет столкновения со всеми платформами
При столкновении с платформой:
Ставит игрока на платформу
Применяет эффект в зависимости от типа платформы
Делает прыжок (vel_y = JUMP_FORCE)
jump()
Заставляет игрока прыгать, если это возможно.
draw(surface)
Рисует персонажа на экране:
Прямоугольное тело
Глаза (два белых круга с черными зрачками)
Анимация покачивания при прыжке

10. КЛАСС PLATFORM (платформа)
python
class Platform:
    def __init__(self, x, y, is_start=False, platform_type=PlatformType.NORMAL):
        self.x = x, self.y = y        # Позиция
        self.width = 100              # Ширина
        self.height = 20              # Высота
        self.platform_type = platform_type  # Тип платформы
        self.is_start = is_start      # Стартовая ли?
        self.move_direction = 1       # Направление движения (для MOVING)
        self.move_speed = 2           # Скорость движения
        self.is_broken = False        # Сломана ли (для FRAGILE)
Методы:

update(player_vel_y)
Если игрок поднимается (vel_y < 0), платформа двигается вниз (создает эффект скроллинга)
Если платформа движущаяся (MOVING), перемещает её влево-вправо

draw(surface)
Рисует платформу:
NORMAL — зеленая
MOVING — оранжевая
FRAGILE — красная с линией посередине
BOUNCE — фиолетовая с белыми точками
ICE — голубая с белой полоской сверху

14. ГЛАВНАЯ ФУНКЦИЯ main()
Это сердце игры. Здесь происходит всё самое важное.

Инициализация (строки 530-545)
python
menu = Menu()                    # Создаем меню
game_state = GameState.MENU      # Начинаем с меню
platforms = None                 # Платформы пока не созданы
player = None                    # Игрок пока не создан
score = 0                        # Счет
high_score = 0                   # Рекорд
Главный игровой цикл (while running:)
В каждой итерации цикла происходит:

1. Обработка событий (event handling)
Закрытие окна (QUIT)
Нажатие ESC — возврат в меню
В меню: обработка нажатий на кнопки
В игре: движение (стрелки/A/D), прыжок (пробел), рестарт (R)

2. Обновление состояния (update)
В меню: menu.update()
В игре:
Обновление позиции игрока
Проверка столкновений
Обновление платформ
Начисление очков
Проверка Game Over

3. Отрисовка (draw)
В меню: menu.draw()
В игре:
Фон (draw_background)
Все платформы
Игрок
Счет, рекорд, подсказки

КАК РАБОТАЕТ ИГРОВОЙ ПРОЦЕСС:
Физика движения:
Каждый кадр к скорости по Y добавляется гравитация
Позиция Y меняется на величину скорости
При столкновении с платформой скорость Y становится отрицательной (прыжок)



Типы платформ и их эффекты:
BOUNCE — прыжок в 1.5 раза выше
ICE — увеличивает горизонтальную скорость
FRAGILE — ломается после двух прыжков
MOVING — движется влево-вправо



ПОТОК ДАННЫХ В ИГРЕ:



Пользователь нажимает кнопку
    ↓
Событие попадает в обработчик
    ↓
Меняется состояние игрока (vel_x, vel_y)
    ↓
В update() применяется физика
    ↓
Проверяются столкновения с платформами
    ↓
Обновляется счет
    ↓
В draw() всё рисуется на экране
    ↓
Экран обновляется (flip)
    ↓
Цикл повторяется 60 раз в секунду

ВАЖНЫЕ ПЕРЕМЕННЫЕ:
game_state — определяет, что сейчас показывает игра (меню, игру, game over)
player.vel_y — скорость по вертикали (положительная = падение, отрицательная = подъем)
player.on_ground — флаг, стоит ли игрок на платформе
platforms — список всех платформ в игре
score — текущие очки
high_score — рекорд за все время



ОСНОВНЫЕ МЕХАНИКИ:
Прыжки — только когда игрок на платформе
Скроллинг — когда игрок поднимается выше середины, платформы двигаются вниз
Респавн платформ — упавшие вниз платформы появляются сверху
Неуязвимость — первые моменты после старта игрок невидим (чтобы не умереть сразу)
Game Over — когда игрок падает ниже экрана
